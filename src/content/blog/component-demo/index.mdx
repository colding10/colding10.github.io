# Introduction

This was a fun crypto challenge from FakeCTF 2026. We're given an RSA encryption with $e = 3$ and a partially known plaintext — classic Coppersmith setup.

The server encrypts `flag + random_padding` under textbook RSA with a tiny public exponent. Since most of the plaintext is known, we can recover the unknown bits using lattice-based methods.

<Callout variant="definition" title="Coppersmith's Theorem">
Let $N$ be an integer and $f(x)$ a monic polynomial of degree $d$. If there exists $x_0 < N^{1/d}$ such that $f(x_0) \equiv 0 \pmod{N}$, then $x_0$ can be found in polynomial time.
</Callout>

The key file is `server.py`:

```python title=server.py
from Crypto.Util.number import getPrime, bytes_to_long

flag = open("flag.txt", "rb").read()
p, q = getPrime(512), getPrime(512)
n = p * q
e = 3

# pad flag with 32 random bytes
import os
padded = flag + os.urandom(32)
m = bytes_to_long(padded)
c = pow(m, e, n)

print(f"{n = }")
print(f"{e = }")
print(f"{c = }")
```

And `output.txt` gives us $n$, $e = 3$, and $c$.

## The Math

We know the flag format is `flag{...}` so we know the upper bits of $m$. Let $m = m_0 + x$ where $m_0$ is the known prefix shifted left, and $x$ is the unknown suffix (the flag content + padding).

Then:

$$
c \equiv (m_0 + x)^3 \pmod{n}
$$

We need to find $x$. Since $x$ is small relative to $n^{1/3}$, Coppersmith's theorem applies directly.

<Callout variant="intuition">
Think of it this way: if $e$ is small and we know most of the message, the "search space" for the unknown part is tiny compared to $n$. Lattice reduction (LLL) finds it efficiently.
</Callout>

## Solution

<Steps>
  <Step title="Parse the output">
    Extract $n$, $e$, and $c$ from `output.txt`. Confirm $e = 3$.
  </Step>
  <Step title="Construct the known prefix">
    We know the flag starts with `flag{`. Convert to an integer and shift left by the number of unknown bits (flag content + 32 bytes of padding).
  </Step>
  <Step title="Build the polynomial">
    Define $f(x) = (m_0 + x)^3 - c$ over $\mathbb{Z}/n\mathbb{Z}$. The small root $x_0$ is our unknown bytes.
  </Step>
  <Step title="Run Coppersmith">
    Use Sage's `small_roots()` to find $x_0$. Recover the full plaintext as $m = m_0 + x_0$.
  </Step>
</Steps>

<Aside title="On epsilon">
The `small_roots()` `epsilon` parameter controls the lattice dimension. Smaller epsilon = better bounds but slower. Start with `epsilon=0.03` and decrease if it doesn't find anything.
</Aside>

n = 0xDEADBEEF  # from output.txt
e = 3
c = 0xCAFEBABE  # from output.txt

# known prefix: "flag{"
prefix = b"flag{"
suffix_len = 100  # unknown bytes (flag body + 32 padding)

m0 = int.from_bytes(prefix, "big") << (suffix_len * 8)

P.<x> = PolynomialRing(Zmod(n))
f = (m0 + x)^e - c

roots = f.small_roots(X=2^(suffix_len*8), epsilon=0.03)
if roots:
    m = m0 + int(roots[0])
    print(int(m).to_bytes(200, "big").strip(b"\x00"))

## Takeaways

<Callout variant="summary">
- RSA with $e = 3$ is dangerous when any part of the plaintext is known or predictable
- Coppersmith's method finds small roots of polynomials mod $N$ in polynomial time
- Sage's `small_roots()` does the heavy lifting — just set up the polynomial correctly
- The bound on the unknown portion must be less than $N^{1/e}$
</Callout>
